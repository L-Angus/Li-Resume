项目背景

K8860 是公司为突破技术封锁、推动国产化替代而自研的通用型 SoC 测试平台，对标 Advantest / Teradyne，填补国产高端 ATE 测试机空白，包含数字、射频、混合信号三大专项，实现全领域测试一站式覆盖。

职责贡献

作为项目初始成员，主导后端系统（ATEStudio）多个核心模块的设计开发，覆盖语言层、调度层、接口层、算法层、数据层以及配置层。

语言层 | ATPL领域特定语言编译器
独立开发

自研ATEStudio的领域专用语言及编译器，用于统一描述测试流程与硬件参数配置。

  • 基于ANTLR4自研DSL语法，以自然语言声明式配置代替C++硬编码。
  • 实现词法→语法→语义三阶段分析，配置错误编译期拦截，精确定位错误位置和类型。
  • 设计 PORT（RETURN/GOTO）流程跳转机制，基于返回码实现条件分支与循环控制。
  • 实现「配置-执行」分离，动态加载立即生效，新业务接入无需修改核心调度代码。
成果： • 效率提升：配置驱动替代硬编码，缩短新品接入周期 • 调试优化：配置变更即时生效，提升调试效率 • 安全防护：编译期拦截配置错误，避免运行时误配风险
调度层 | 多DUT并行测试流调度引擎
独立开发

自研ATEStudio的核心调度引擎，基于ATPL解析数据，驱动多DUT并行测试执行和调度。

  • 基于 DFS 构建父子依赖关系，完成环路检测与合法性校验，输出可执行任务图。
  • 基于状态机与依赖驱动机制完成任务分发，利用反射工厂实现测试插件热加载。
  • 设计 Port 分支路由机制，各 DUT 按结果驱动 GOTO 流程跳转/RETURN 结果回传。
  • 支持循环语义，循环调度与主调度器解耦，结果经调度器栈中转归并。
  • 设计监听器机制，调度器专注流程控制，附加功能（断点/上报/分Bin）外部扩展。
成果： • 调度能力：复杂流程稳定执行，支持多DUT多场景统一调度 • 扩展优化：测试插件热加载，新增业务测试场景无需修改调度核心 • 性能安全：静态构图+动态执行分离，流程错误前置拦截，运行效率显著提升
接口层 | SDK架构分层和模块化设计
独立开发

作为ATEStudio中间层，主导SDK架构分层与模块化设计，拉通上下游端到端链路闭环。

  • 作为中间层，采用四层分层架构（RPC→接口层→实现层→硬件抽象层），实现上位业务与下位硬件解耦。
  • 作为后端，提供 RPC 方法，承接前端 UI 调度指令，路由执行链路并回传结果。
  • 作为 SDK，主导业务模块链式调用接口开发，支持用户二次开发测试多种业务场景。
  • 作为上位机，封装上位业务逻辑交叉编译为 .so 库，动态注入下位 RPC 服务，实现自主可控的硬件驱动。
成果： • 开发效率：链式调用与反射机制支撑用户二次开发，降低业务接入成本 • 业务覆盖：支撑数字/射频/混合多业务接入，复用同一套SDK框架 • 端到端闭环：形成"前端触发→SDK调度→用例执行→接口调用→硬件驱动"完整链路闭环
算法层 | 高精度DSP信号分析系统
独立开发

自研ATEStudio的DSP信号分析系统，服务于射频、MIX专项ADC数据的分析和处理。

  • 从理论调研到工程化落地，实现多种 DSP 算法，构建数据采集→加窗处理→FFT转换→频谱分析→指标计算完整 Pipeline。
  • 实现 5 种窗函数（矩形/汉宁/汉明/布莱克曼/平顶），有效抑制频谱泄露。
  • 基于 FFTW 实现 FFT 引擎，引入 Coherent Gain+ENBW 补偿模型，精度误差控制在 0.01dBm 以内。
  • 基于动态直方图法，实现 DNL/INL 分析，覆盖高精/高速 ADC 线性度测试。
  • 实现 SNR/SND/THD/SFDR 指标计算，支持结构化输出功率分解结果。
成果： • 效率提升：填补测试平台信号分析能力空白，提高产品自动化效率 • 精度优化：频谱分析精度 < 0.01dBm（对比频谱分析仪/VSA89600） • 成本节约：降低第三方商业软件依赖，节省昂贵授权成本
数据层 | 异步STDF数据记录系统
独立开发

自研ATEStudio的数据记录系统，可生成通用标准STDF文件，支持跨平台数据分析与芯片良率评估。

  • 基于 STDF 协议规范，实现 17 种记录类型与二进制序列化接口，覆盖测试全生命周期。
  • 采用生产者-消费者模式，主线程生成记录，工作线程异步写入，实现测试与 IO 解耦。
  • 设计对象池管理 Record 对象，减少高频场景内存分配开销。
  • 设计背压监控机制，实时监测队列积压，超过阈值触发告警，防止内存溢出。
  • 设计熔断机制，支持正常完成、异常停止、超时强制停止，保障系统稳定性。
成果： • 跨平台性：生成行业标准格式STDF文件，支撑下游良率分析。 • 效率保障：异步写入与主流程解耦，兼顾数据完整性与测试效率。 • 运行保障：支撑7×24h零故障运行与千万级记录写入，保障量产数据完整可追溯。
配置层 | 高性能CSV解析库
独立开发

自研轻量级高性能CSV解析组件，支撑ATEStudio核心业务模块的配置解析。

  • 策略模式封装同步/异步解析策略，支持运行时动态切换，业务层按需接入。
  • 采用 std::string_view 实现全链路零拷贝，减少堆内存分配开销。
  • 基于 atomic 实现无锁"取号机"任务分发模型，充分发挥多核性能，实现动态负载均衡。
  • 结合 RAII + 智能指针统一管理文件 IO，保障资源异常安全、无泄漏。
  • 设计回调机制，支持业务逻辑自定义扩展，适配多样化配置解析需求。
成果： • 解析高效：异步模式充分利用多核并行性能，1000w行解析1.9s。 • 轻量复用：轻量化设计，零第三方库依赖，控制打包体积与部署成本。 • 平台价值：纳入平台基础组件，支撑ATEStudio多个核心业务配置解析。
